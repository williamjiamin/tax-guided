## What this project is

**taxguided.com** — a global tax content site covering tax laws, regulations, news, case law, treaties, statutes, and reference data across multiple jurisdictions. Currently strongest in U.S. + China; expanding.

The product is **content-as-product**: the site IS the value, monetized through SEO traffic + (planned) subscription tiers. Comparable in shape to NerdWallet / Investopedia / Tax Foundation, narrower in scope to tax + adjacent topics.

This is a **YMYL** (Your Money or Your Life) site under Google's Quality Rater Guidelines — the E-E-A-T bar is high. Anything published affects readers' tax positions.

## Cross-project router (NEW — important)

The sister project `foreignllctax.com` runs a daily cron at 04:17 UTC that fetches IRS Newsroom + Federal Register tax-relevant items, applies a foreign-LLC-specific filter, and:

- Items relevant to foreign-owned U.S. LLC owners → consumed by foreignllctax's own Q&A pipeline
- Items NOT foreign-LLC-relevant but still generally tax-relevant → **routed to taxguided.com via a public JSON feed**
- Items truly off-topic (advocacy panel meetings, dyed fuel refunds, etc.) → silently dropped

### What you (the tax-guided AI session) should build to consume it

A daily GitHub Actions workflow that:

1. Fetches `https://foreignllctax.com/tax-news-router-output.json`
2. Diffs against `.cron/news-seen.json` (committed state file in this repo)
3. For each new item, decides whether it fits an existing tax-guided content type:
   - Regulation → append to `data/regulations.latest.json` + regenerate `src/lib/regulations/feed-data.ts`
   - News article → write to a new `src/lib/news/auto-news.ts` (mirror foreignllctax's drip pattern)
   - Statute/treaty/case-law update → route to the appropriate lib
4. Auto-commits + pushes (or opens a PR — your call based on confidence)

## Don't repeat foreignllctax's mistakes

These are documented in foreignllctax's `docs/qa-architecture.md` and `docs/seo-lessons.md`:

1. **Worker self-fetch loops cause HTTP 522.** If you need a worker to read its own data, use `env.ASSETS.fetch()` for static assets or pre-bake a JSON file at build time. Do not `fetch("https://taxguided.com/...")` from inside the worker.
2. **AI hallucinates relevance under auto-merge.** Always combine: (a) pre-AI keyword filter that rejects obvious off-topic items, (b) AI prompt with explicit `relevanceScore` integer field, (c) hard validation that rejects scores below 7. Without all three, the model will invent fake relevance for fishing-rights regulations and similar.
3. **Don't bundle large content data into the runtime worker.** Cloudflare Workers cap at 10 MiB compressed. Move content to `.snapshot/` JSON files (mirror foreignllctax's `.qa-snapshot/` and `.articles-snapshot/` patterns).
4. **Don't dump 1000+ URLs into the sitemap at once.** Drip new content (max 5-10/day). Google interprets sudden sitemap explosions as low-quality-batch and slows crawl rate.
5. **Don't skip staging deploy.** Set up a `wrangler.staging.toml` and deploy there first. foreignllctax had two visitor outages from skipping this.
6. **Don't add `force-dynamic` or remove `generateStaticParams` from cacheable routes.** Cache-control `no-cache, no-store` is the most expensive header on a high-traffic site.
